# Phase 06: Coding Practice IDE And Runner

Status: planned.

## Goal

Build a LeetCode-like practice loop with AI mentoring:

1. generate or select a coding task;
2. solve it in a web editor;
3. run deterministic tests;
4. request hints when stuck;
5. submit;
6. receive AI code review and score adjusted for hints;
7. store progress evidence.

## User Outcome

The user can practice interview coding tasks locally and receive both test feedback and mentor-style code review.


UI areas:

- task statement;
- examples;
- editor;
- language selector;
- run button;
- submit button;
- test output;
- hints panel;
- review panel.

Mobile:

- task/editor/results should stack;
- editor must have stable height;
- long output should scroll inside a bounded area.

## Runner Work

Implement runner abstraction first:

```txt
packages/code-runner/src/
  runner.ts
  languages/
    javascript-runner.ts
    python-runner.ts
  tests/
    test-case.ts
    result-normalizer.ts
```

MVP language choice:

- start with JavaScript/TypeScript or Python;
- do not try to support all languages at once.

Runner must provide:

- timeout;
- stdout;
- stderr;
- duration;
- pass/fail;
- per-test result;
- normalized runtime error.

## AI Mentor Work

AI responsibilities:

- generate coding task from skill and goal;
- generate progressive hints;
- evaluate final solution quality;
- explain failed tests;
- assign score with hint penalty.

Score should consider:

- correctness;
- complexity;
- readability;
- edge cases;
- number and depth of hints used;
- whether the user needed direct solution-level help.

## Verification

Required:

```powershell
pnpm typecheck
pnpm build
```

Manual:

- open a coding task;
- edit code;
- run tests;
- see test output;
- request a hint;
- submit solution;
- see review;
- confirm attempt/evidence persisted.

Security verification:

- runner enforces timeout;
- runner captures output;
- runner does not execute unrestricted host commands.

## Risks

- Unsafe execution. This is the highest-risk subsystem.
- AI review replacing tests. Tests must be first.
- Too many languages too early. Start narrow.
- WebContainers may impose product constraints; evaluate before relying on them.

## Done Criteria

- One language works end-to-end.
- Deterministic test results are stored.
- Hints are tracked and affect score.
- AI review produces structured evaluation.
- Coding evidence updates skill progress.
